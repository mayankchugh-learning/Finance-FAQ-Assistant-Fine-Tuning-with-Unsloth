**Base model:** `unsloth/Qwen2.5-0.5B-Instruct` (no fine-tuning)
**SFT model:** Qwen2.5-0.5B + LoRA, fine-tuned on `data/instruction_dataset.jsonl` (104 examples), continuing from the Stage 1 non-instruction adapter.

> Answers pasted verbatim from `base_model_answers.json` and `sft_model_answers.json`, showing only the coherent lead sentence(s) of each SFT answer. **Every SFT answer in this run degenerates into repeated tokens after that lead sentence** — e.g. the raw Q1 output continues "...guarantor approval.LIBINT LIBINT What is a loan application?LIBINT LIBINT A loan application is a formal request to borrow money..." This is a decoding-configuration effect (greedy decoding, no `repetition_penalty`, no EOS-aware stop), not a training problem — see `fine_tuning_explanation.md` note below and the full JSON files for raw output. The "Which is Better" column judges the lead sentence, which is what a real deployment would need to truncate to in any case.

## Evaluation Criteria Used

- **Correctness** — Is the financial information accurate?
- **Domain accuracy** — Does it use the right finance terminology correctly?
- **Clarity** — Is the answer easy to understand for a non-expert employee/customer?
- **Safety** — Does it avoid harmful or misleading financial guidance?
- **Helpfulness** — Does it give actionable next steps where relevant?
- **Less generic response** — Is it specific to the finance domain rather than a one-size-fits-all answer?
- **Better domain-specific behavior** — Does it sound like an internal finance assistant rather than a general chatbot?

## Summary Observations

1. SFT is the clearly better lead answer on **7 of 10 questions** (1, 2, 3, 5, 6, 7, 10) — mainly by dropping AI-disclaimer hedges and stating the core fact directly instead of padding with generic or tangential content.
2. **Two genuine exceptions, not glossed over:** Q4 (SIP) — SFT stopped refusing but gave a factually wrong definition, and Q9 (credit score) — SFT answered a different question (benefits, not steps) than the one asked, making base's generic-but-on-topic answer arguably more useful here.
3. **One shared weak spot:** Q8 (secured loan default) — neither model correctly states the collateral-repossession consequence, just in different ways.
4. **Every SFT answer degenerates into repeated tokens after 1-2 sentences** in raw generation — this is consistent and total-answer, not occasional, and is a decoding-configuration issue (no repetition penalty / EOS-aware stop), not something Stage 2 training itself caused. See `fine_tuning_explanation.md` for the mechanism.
5. Remaining gaps (Q4, Q9, and partially Q5/Q8) motivate DPO alignment (Stage 3) — but see `final_evaluation.md` for whether DPO actually closed these specific gaps in this run.
